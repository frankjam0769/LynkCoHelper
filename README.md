```
1.config.json
该文件存放应用基本配置信息，无特殊情况请不要尝试修改
2.account.json
该文件存放用户账号密码信息，启动脚本前请在该文件配置账号密码

3.已知缺陷
由于APP端登录采用单点登录方式限制，脚本每运行一次，对应账号的Token就会失效，对于会经常使用APP端的人来说，就需要重新登录一次，因为脚本是每天跑一次，故每天都会需要重新登录一次APP

```


# 使用方法

```
1.打开[项目地址](https://github.com/a437936609/LynkCoHelper)，点击右边的Fork，Fork到自己的账号（不放心账号密码安全的，建议克隆项目，自己建立私仓）

2.到自己Fork后的项目，修改LynkCoHelper下的account.json文件，填写自己的领克APP账户密码，填写完点击保存

3.到自己Fork的项目，点击Actions，然后启用Workflows，启动后会在每天凌晨的6时.30分执行（UTC+8时间）
```
